# Super-Resolution: реализация метода Максимова

Локальная версия ноутбука. Сравнивает классическое многокадровое
суперразрешение (по диссертации Максимова) с предобученной и дообученной
нейронной сетью EDSR.

## Установка

```bash
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
jupyter notebook SRW.ipynb
```

PyTorch с поддержкой GPU при необходимости ставится отдельно по инструкции на
[pytorch.org](https://pytorch.org/get-started/locally/) — выбор зависит от
версии CUDA. Без GPU всё тоже работает (только медленнее).

## Структура

```
.
├── SRW.ipynb               этот ноутбук
├── requirements.txt
├── data/
│   ├── test/base.png       положите сюда эталонное HR-изображение
│   └── DIV2K/              датасет, скачивается автоматически из ноутбука
├── models/                 обученные веса (.pth)
└── results/
    ├── figures/            PNG всех графиков
    └── cache/              pickle с промежуточными данными
```

Перед первым запуском нужно положить эталонную картинку в `data/test/base.png`.

## Кеширование

Все долгие шаги (генерация LR-кадров, регистрация, обучение) сохраняют
результат на диск. При повторном запуске данные подгружаются из
`results/cache/`, а обученная модель — из `models/edsr_finetuned.pth`.

Чтобы пересчитать какой-то этап, удалите соответствующий файл:

| Шаг                        | Что удалить                                      |
|----------------------------|--------------------------------------------------|
| Генерация LR-кадров        | `results/cache/lr_frames.pkl`                    |
| Регистрация кадров         | `results/cache/aligned_frames.pkl`               |
| Обучение                   | `models/edsr_finetuned.pth` + `results/cache/training_history.json` |

Графики сохраняются в `results/figures/` автоматически при каждом запуске
соответствующих ячеек — их можно использовать в отчёте, не запуская ноутбук.
